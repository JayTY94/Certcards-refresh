Selection vs Range (Word VBA)
Selection represents the user’s current cursor position or highlighted content and changes as the cursor moves. Range is an object that represents a portion of a document and can be manipulated without moving the cursor, making Range safer and more stable for Word automation.
........
ActiveDocument.UpdateStyles
A Word VBA method that updates the styles in the active document to match those stored in the attached template (such as Normal.dotm). This is commonly required before applying template-defined styles to documents created from other sources.
........
Application.OrganizerCopy
A Word VBA method used to copy styles, macros, or other objects between documents or templates. It is often used to copy required styles from Normal.dotm into the active document when those styles do not already exist.
........
Style for Following Paragraph
A Word style property that determines which style is automatically applied when the user presses Enter after a paragraph. When styles are chained, dependent styles must exist in the document for the chain to work correctly.
........
Normal.dotm
The global Word template that loads at application startup. Macros and styles stored in Normal.dotm are available to all documents for that user, making it the execution boundary for global Word macros.
........
Global Word Template Deployment
A deployment approach where a customized macro-enabled template replaces the default Normal.dotm so that firm-wide macros and styles are automatically available in every Word session.
........
Link to Previous (Headers and Footers)
A section-level setting in Word that causes a header or footer to inherit content from the previous section. If enabled, changes to one section’s header or footer will propagate to linked sections.
........
Word Sections
Logical divisions within a Word document that allow different page layouts, headers, footers, and formatting rules. Headers and footers are scoped to sections, not the entire document.
........
Selection.Type
A Word VBA property that indicates the type of current selection, such as normal text, insertion point, table, shape, or header/footer. Checking Selection.Type helps prevent runtime errors when macros assume text selection.
........
wdSelectionNormal
A Selection.Type constant indicating that the selection is normal text. Macros that format text often check for this value before proceeding.
........
wdSelectionIP
A Selection.Type constant indicating an insertion point with no highlighted text. Often treated similarly to normal text selection when applying paragraph-level formatting.
........
Paragraph Object (Word VBA)
Represents a paragraph in a Word document. Looping through ActiveDocument.Paragraphs is the safest way to apply formatting consistently across all paragraphs.
........
Recorded Macro Refactoring
The process of converting recorded Word macros that rely heavily on Select and Activate into production-ready code using explicit Range and object references.
........
Cursor Jumping in Word Macros
An undesirable behavior caused by excessive use of Selection.Move, Select, or Activate. It is avoided by performing formatting operations on Range objects instead of Selection.
........
Template-Dependent Styles
Styles that exist in a template (such as Normal.dotm) but may not exist in documents created elsewhere. Macros must ensure these styles are imported or updated before applying them.
........
Header/Footer Scope
In Word, headers and footers belong to sections rather than the document as a whole. Modifying headers or footers without accounting for section boundaries can cause unintended changes.
........
Defensive Word VBA Coding
A coding approach that validates selection type, document state, and style availability before applying formatting, reducing runtime errors and unpredictable behavior.
........
Range.Expand wdParagraph
A Word VBA operation that expands a Range to include the entire paragraph containing it. Useful for applying paragraph-level formatting without moving the cursor.
........
ScreenUpdating (Word VBA)
An Application-level property that controls whether Word visually updates the screen during macro execution. Disabling it improves performance but does not fix logical issues caused by Selection misuse.
........
Style Does Not Exist Error
A common Word VBA runtime error that occurs when a macro attempts to apply a style that is not present in the active document’s Styles collection.
........
Production-Ready Word Formatting Macro
A macro that uses Range instead of Selection, validates Selection.Type, ensures required styles exist, and respects section and header/footer boundaries.

Sub … End Sub
A procedure block that runs actions but does not return a value. VBA uses Sub to mark the start of the macro and End Sub to mark the end. [VBA Coding...s for Word | Word]
........
Function … End Function
A procedure block that returns a value to its caller. Functions can be used in code and (in Excel) often in worksheet cells, whereas Word macros are commonly written as Subs.
........
Comments ( ' )
Any text after a single quote is ignored by VBA at runtime. Comments are used to document intent and clarify what each section of a macro does. [VBA Coding...s for Word | Word]
........
Line Continuation ( _ )
An underscore preceded by a space tells VBA that a statement continues on the next line. It is commonly used to keep long argument lists readable, especially in Word methods that take many named parameters. [How to Cre...ORD Macros | Word]
........
Set (Object Assignment)
The Set keyword assigns an object reference to an object variable. Without Set, VBA attempts value assignment and will error when working with objects.
........
Variant Type
A flexible VBA type that can hold different kinds of values (string, number, date, object reference). Variants make quick macros easy but can hide type issues unless you validate and convert intentionally.
........
ByRef
A parameter passing mode where the called procedure can modify the caller’s variable directly. This is the VBA default if you do not specify ByVal.
........
ByVal
A parameter passing mode where the procedure receives a copy of the value. Changes inside the procedure do not modify the caller’s original variable.
........
With … End With
A VBA block that lets you set multiple properties or call multiple methods on the same object without repeating the object name. It starts with With <object> and ends with End With. [VBA Coding...s for Word | Word]
........
Dot Prefix Inside With ( .Property / .Method )
Within a With block, a leading dot means “use the With object.” This avoids repeating the full object path and reduces errors when editing repeated references. [VBA Coding...s for Word | Word]
........
If … Then … Else
A conditional control structure that runs one block of code when a condition is True and another block when it is False. It is the most common branching structure in VBA.
........
Select Case
A multi-branch control structure that chooses one of many paths based on a single expression. It is often clearer than many ElseIf blocks when you have multiple discrete cases.
........
For … Next
A counting loop that repeats a block a specified number of times. It is typically used when you need an index (i) and known bounds.
........
For Each … Next
An iteration loop that visits each item in a collection (e.g., paragraphs, sections, shapes). It is usually safer than index loops for Word collections.
........
Do While / Do Until
A loop that repeats while a condition is True (Do While) or until a condition becomes True (Do Until). It is used when you don’t know the exact number of iterations ahead of time.
........
Exit Sub / Exit Function
Early termination statements that immediately leave a procedure. They are commonly used for defensive checks (e.g., “nothing selected → exit”) and for clean control flow after successful completion.
........
GoTo label
A branching statement that transfers control to a labeled line within the same procedure. It cannot jump into or out of a procedure or into/out of a With block. [12754 | PDF], [proggd | PDF]
........
Label (e.g., ErrorHandler: )
A named target within a procedure used by GoTo or error handling constructs. Labels exist only within the procedure where they are declared. [proggd | PDF]
........
On Error GoTo Handler
A VBA error-handling pattern that redirects execution to a labeled handler when a runtime error occurs. It is commonly paired with cleanup logic and a single exit path. [12754 | PDF]
........
On Error Resume Next
A VBA error-handling mode that tells VBA to continue execution on the next statement after an error. It can be useful for “try-and-check” patterns but is risky if you do not validate Err.Number and reset error handling.
........
Err Object
A global VBA object that holds details about the most recent runtime error (number, description). It is typically checked after On Error Resume Next and cleared with Err.Clear.
........
Named Arguments
A VBA calling style that specifies parameter names (e.g., Source:=... Destination:=...). It improves readability and reduces mistakes when calling methods with many parameters. [How to Cre...ORD Macros | Word]
........
Parentheses in Procedure Calls
In VBA, parentheses are required when you use Call or when passing arguments to a Function whose return value is being used. Misplacing parentheses can change evaluation order and cause confusing errors.
........
String Concatenation ( & )
The & operator joins strings reliably. It is preferred over + because + can attempt numeric addition and produce type coercion surprises.
........
Implicit Type Suffixes (%, $, @)
Legacy VBA type characters that force a variable’s type (e.g., % for Integer). They can reduce clarity and are generally avoided in modern VBA style, but you will still see them in older codebases. [12754 | PDF], [proggd | PDF]
........
Public vs Private (Scope)
Public makes a procedure or variable accessible outside the module, while Private limits it to the module. Scope affects how macros are exposed and how naming conflicts occur in large macro collections.
........
Module vs Procedure Scope
A variable declared at the top of a module persists for the lifetime of the host application session, while a variable declared inside a Sub or Function exists only during that procedure’s execution.
........
Object vs Value Default Properties
Some VBA objects expose default properties that VBA may use implicitly if you omit the property name. Relying on defaults can make code shorter but less readable and more error-prone.
........
Type Mismatch
A runtime error that occurs when VBA attempts to use a value as a different type than it actually is (e.g., converting a blank or non-numeric string to an Integer). Defensive conversion patterns prevent this category of failures.
