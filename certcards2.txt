13
SSH + SCP
PEM vs OpenSSH Keys
PEM is a generic container often used for TLS/SSH keys (.pem). OpenSSH can read PEM and its own modern OpenSSH format. Cloud tools sometimes expect PEM; PuTTY expects PPK.





14
SSH + SCP
Passphrase vs Password (SSH)
A passphrase secures the private key file itself. A password is an account credential used for interactive login. With key auth, you usually don’t need the account password.





15
SSH + SCP
ED25519 vs RSA
ED25519: modern, fast, small keys, strong security with short fingerprints.
RSA: widely compatible; larger keys; still common but ED25519 preferred when supported.





16
SSH + SCP
SCP vs SFTP vs rsync
SCP: simple copy over SSH; minimal features.
SFTP: SSH file transfer protocol; resumes, remote listing, better tooling.
rsync: efficient delta copy, preserves metadata; can use SSH as transport.





17
SSH + SCP
Ciphers and MACs (SSH)
Ciphers like aes128-gcm or chacha20‑poly1305 encrypt data; MACs (or AEAD modes) ensure integrity. Prefer modern AEAD ciphers for speed and security.





18
SSH + SCP
authorized_keys
A file on the destination host that lists allowed public keys (one per line) for a user. OpenSSH reads it during login; incorrect permissions or ownership will cause key rejection.





19
SSH + SCP
Key Exchange (KEX)
Algorithms used during SSH handshake to derive shared secrets securely (e.g., curve25519‑sha256). Strong KEX protects against interception during session setup.





20
SSH + SCP
Public Key Authentication (SSH)
A method where the client proves identity using a private key while the server checks the corresponding public key in ~/.ssh/authorized_keys. No password is sent; authentication succeeds if the key pair matches and permissions are correct.





21
Binary File Analysis
Binary Carving
Binary carving is the practice of scanning raw bytes for recognizable patterns (strings, integers, repetition) to infer structure when no schema is available.





22
Binary File Analysis
Endianness
Endianness describes the byte order used to store multi‑byte values in memory. Little‑endian systems store the least significant byte first (common on x86), while big‑endian systems store the most significant byte first. Misunderstanding endianness leads to reading correct bytes but interpreting the wrong number.





23
Binary File Analysis
ASCII‑Printable Range
The ASCII‑printable range (0x20–0x7E) represents visible characters. String carving relies on contiguous bytes in this range, which fails when padding or encoding breaks contiguity.





24
Binary File Analysis
UTF‑16LE
UTF‑16LE encodes characters as two bytes (character byte + 0x00). This encoding breaks simple ASCII string carving and requires different detection logic.





25
Binary File Analysis
Byte Offset
A byte offset is the zero‑based position of a byte within a file or memory buffer. Offsets are how record‑oriented systems locate fields without delimiters, using fixed positions instead of searching.





26
Binary File Analysis
Page
A page is a fixed‑size block of data (commonly 4 KB) used by the operating system and database engines for I/O and caching. Files are often processed page‑by‑page, not record‑by‑record.





27
Grammar
Elliptical Clause
An elliptical clause omits material that is understood from context. For example, “She reviewed the logs, and he did too” omits reviewed the logs in the second clause. Ellipsis is grammatical and common in coordinated and comparative structures, but the missing elements must be recoverable.





28
Grammar
Genitive Case
The genitive case marks possession, association, or close relationship, typically using apostrophe‑s. For example, “The committee’s decision” does not imply ownership in a literal sense but indicates association. The genitive case is broader than simple possession and applies to abstract relationships as well.





29
Grammar
Non‑Restrictive Relative Clause
A non‑restrictive relative clause adds supplementary information about a noun that is already identified. It is set off by commas and can be removed without changing the core meaning of the sentence. For example, “The server, which was recently upgraded, restarted overnight.” These clauses provide commentary rather than definition.





30
Grammar
Subordinate Clause
A subordinate clause is any dependent clause that is grammatically subordinate to another clause. This includes adverbial, relative, and noun clauses. Subordination signals hierarchy in sentence structure, indicating that one clause provides context or content for another rather than equal status.





31
Grammar
Concessive Clause
A concessive clause is a subtype of adverbial clause that expresses contrast or unexpected outcome. It acknowledges a condition that might normally prevent the main clause but does not. Common markers include although, though, while, and even though. For example, “Although the solution was elegant, it was impractical.” The key feature is logical contrast, not opposition of actions.





32
Grammar
Participle / Participial Phrase
A participle is a verb form (usually ‑ing or ‑ed) used as an adjective to modify a noun. A participial phrase includes the participle plus any accompanying modifiers or objects. For example, in “Walking down the street, she noticed the buildings,” walking down the street modifies she. Participial phrases must attach logically to the noun they describe.





33
Grammar
Argument (Verb Complement)
An argument is a sentence element required by a verb to complete its meaning. Arguments are part of the verb’s core structure and cannot be removed without making the sentence incomplete or ungrammatical. For example, rely requires a complement: “She relied on her experience.” Removing on her experience leaves the verb’s meaning unfinished.





34
Grammar
Infinitive of Purpose
An infinitive of purpose explains why an action was taken. It usually answers the question “why?” For example, “He stopped to smoke” means he stopped in order to smoke. These infinitives are optional and describe intent rather than completing the verb’s meaning.





35
Grammar
Relative Clause
A relative clause modifies a noun and provides additional information about it. It is typically introduced by a relative pronoun such as who, which, that, or whom. For example, in “The report that you sent was incomplete,” that you sent modifies report. Relative clauses function adjectivally and are embedded within noun phrases.





36
Grammar
Noun Clause (That‑Clause)
A noun clause functions syntactically as a noun and commonly begins with that. It can act as a subject or object. For example, “That the system failed was obvious” uses a noun clause as the subject. In many cases, the word that can be omitted without changing the clause’s function, especially when the clause is an object.





37
Grammar
Subjunctive Mood
The subjunctive mood expresses necessity, recommendation, demand, or hypothetical conditions rather than factual statements. In modern English it appears mainly in that‑clauses after certain triggers, such as important, necessary, essential, recommend, or demand. For example, “It is essential that she be present.” The verb uses its base form, not standard agreement.





38
Grammar
Coordinate Clause
A coordinate clause is an independent clause joined to another independent clause using a coordinating conjunction such as and, but, or or. For example, “The test failed, and the deployment was halted.” Neither clause depends on the other, and both could stand alone as sentences.





39
Grammar
Object Clause
An object clause is a noun clause that functions as the object of a verb. For example, “She confirmed that the system failed.” The clause completes the meaning of the verb confirmed. Object clauses are arguments, not modifiers, and cannot usually be removed without affecting grammaticality.





40
Observability and SRE
Latent Conditions are hidden weaknesses in a system that do not cause failures on their own but contribute to incidents when combined with other factors. These conditions often accumulate silently and are a key focus of blameless postmortems.





41
Observability and SRE
Signal vs Context
A signal directly indicates system correctness or user experience, such as error rate or request success. Contextual metrics, like traffic volume or CPU usage, provide background information but are not sufficient on their own to determine whether a system is failing.





42
Observability and SRE
Time-weighted failures recognize that frequent, intermittent failures often degrade user trust more than a single sustained outage. As a result, many short incidents can be more damaging to reliability objectives than one longer failure.





43
Observability and SRE
Impact-based Alerting vs Symptom-based Alerting
Impact-based alerting triggers when users are actually experiencing problems, such as elevated error rates or failed requests. Symptom-based alerting triggers on internal signals like CPU usage, memory pressure, or pod restarts that may correlate with failures but do not necessarily indicate user impact. Mature SRE practices prefer impact-based alerts for paging to reduce alert fatigue.





44
Observability and SRE
Blast Radius describes the scope of impact when a failure occurs, including how many users, services, or regions are affected. Observability practices focus on rapidly determining blast radius to guide response priority and communication.





45
Observability and SRE
Time-weighted Reliability means that when failures occur matters, not just how many occur. Multiple small outages spread over time can consume error budgets faster than a single larger outage, which is why frequent intermittent failures are treated as higher risk in SRE practices.





46
Observability and SRE
An Unknown-Unknown is a failure mode that was not anticipated and therefore has no predefined alert or dashboard. Observability is designed to support investigation of unknown-unknowns by enabling flexible querying and correlation rather than relying solely on predefined checks.

Trusted Launch vs. Confidential VM 
Trusted Launch protects the boot chain (rootkits, tampered firmware) but the host hypervisor can still read VM memory. 
Confidential VM adds hardware memory encryption so even Microsoft hosts can't peek at RAM. 
Use Trusted Launch as the default; Confidential VM when "even the cloud provider is in your threat model."





254
Azure Compute Gallery and Disk Architecture
NVMe vs. SCSI — why NVMe wins on modern hardware 
SCSI was designed for single-head spinning disks: one command queue made sense. 
NVMe was designed for parallel SSDs from scratch: up to 65,535 queues, each up to 65,535 commands deep. On many-core CPUs, every core can run its own queue without contending through one bottleneck. The result is dramatically higher IOPS-per-vCPU.

    Install software, configure, run any tests
    Run sysprep /generalize
    Capture to a managed image (intermediate) or directly to a gallery version
    Tear down the temp resource group
    The transient resource group is owned and cleaned up by Packer — don't put anything else in it.

    Gen 1 = Legacy BIOS + MBR.

Grammar
Dependent (Subordinate) Clause
A dependent clause contains a subject and a verb but cannot stand alone as a complete sentence. It depends on an independent clause to supply full meaning. Dependent clauses are often introduced by subordinating conjunctions such as because, although, while, if, or that. For example, “because the system failed” must attach to another clause to be grammatical.





290
Grammar
Lexical (Action) Verb
A lexical verb expresses an action, process, or event performed by the subject. Unlike copular verbs, lexical verbs contribute concrete meaning to what happens in the sentence. In “She tasted the soup,” tasted is a lexical verb describing an action. Many verbs can function either lexically or copularly depending on how they are used.





291
Grammar
Restrictive Relative Clause
A restrictive relative clause narrows or defines which specific noun is being referred to. It is essential to the sentence’s meaning and is not set off by commas. For example, “Users who reset their passwords must reauthenticate” implies that only a subset of users is being discussed. Removing the clause changes the meaning of the sentence.





292
Grammar
Free (Headless) Relative Clause
A free relative clause functions as a noun phrase even though it looks like a relative clause. It has no explicit antecedent noun. For example, in “I know what she said,” what she said means “the thing that she said.” These clauses often begin with what, whoever, whatever, or wherever.





293
Grammar
Adjectival “Be” (Stative Construction)
Not all be + past participle constructions are passive. Some describe a state rather than an action. For example, “The window was broken and jagged” describes the window’s condition, not the act of breaking it. These constructions lack an implied agent and are adjectival rather than passive.





294
Grammar
Restrictive Relative Clause
A restrictive relative clause provides essential information that narrows the meaning of a noun. It is not set off by commas because removing it would change the sentence’s meaning. For example, “Students who studied passed” implies that only some students passed. Restrictive clauses define which specific noun is being discussed.





295
Grammar
Embedded Interrogative Clause
An embedded interrogative clause reports a question rather than asking one directly. It often looks identical to a free relative clause but serves a different function. For example, “I asked what she said” reports a question, while “I know what she said” states knowledge. The difference lies in the governing verb (ask vs know).





296
Grammar
Adjectival Clause
An adjectival clause modifies a noun, functioning similarly to an adjective. Most adjectival clauses are relative clauses, but participial clauses also fall into this category. For example, “The user logged in using MFA” contains an adjectival participial clause modifying user. These clauses attach directly to noun phrases.





297
Grammar
Gerund (Gerund Phrase)
A gerund is the ‑ing form of a verb that functions as a noun. Gerunds can act as subjects, objects, or objects of prepositions. For example, in “Running late annoyed everyone,” running late is a gerund phrase functioning as the subject of the sentence. A common test is whether the phrase could be replaced by a noun like something without changing the sentence’s structure.

A participial clause uses a participle (‑ing or ‑ed) and functions adjectivally. For example, “Running late, he skipped the meeting.” Participial clauses do not have an explicit subject and must logically attach to the subject of the main clause. When they do not, the result is a dangling participle.

A subject clause is a noun clause that occupies the subject position of a sentence. Because long clauses can be heavy, English often delays them using anticipatory it. An adjunct is optional information that adds detail such as time, place, manner, or reason. Adjuncts are not required by the verb and can usually be removed without breaking grammaticality. MCP security risks (the real ones)
Prompt injection — a malicious resource or tool description injects instructions the model then follows.
Tool combination exfiltration — combining permissions across tools to leak data in ways no single tool would allow.
Lookalike tools — a malicious server registers a tool with a name/description nearly identical to a trusted one, silently substituting itself.
The spec relies on user consent at every step because the protocol can't enforce trust. Hosts must show what's being called, with what data, and require approval — especially for destructiveHint tools and sampling requests.

Elicitation — server asks the user (via the host UI) for additional input mid-operation.
These flow from server to host — the reverse of the usual direction.

543
Grammar
Infinitival Complement
An infinitival complement is an infinitive required by a verb to complete its meaning. In “I told him to leave early,” to leave early is required by told. Removing it makes the sentence incomplete. Unlike infinitives of purpose, complements are arguments, not modifiers.





544
Grammar
Embedded Interrogative Clause
An embedded interrogative clause reports a question rather than asking one directly. It typically appears after verbs like ask, wonder, know, or remember. For example, “I asked what she said” embeds a question inside a declarative sentence. Unlike direct questions, embedded interrogatives do not use subject‑auxiliary inversion.





545
Grammar
Infinitival Clause
An infinitival clause uses the to + verb form and can function as a complement or a modifier. For example, “She decided to postpone the deployment” uses an infinitival clause as a complement of decided. Infinitival clauses can also express purpose, as in “She paused to think.”





546
Grammar
Dangling Participle
A dangling participle occurs when a participial phrase has no clear or logical noun to modify. This creates an unintended or nonsensical meaning. For example, “Walking down the street, the buildings looked familiar” implies that the buildings were walking. Dangling participles are grammatical errors caused by missing or misplaced subjects.





547
Grammar
Object Complement
An object complement follows a direct object and provides additional information that renames or describes that object. It commonly appears with verbs like make, call, name, elect, or consider. In “They elected her president,” president is an object complement describing her. Object complements are neither modifiers nor indirect objects.





548
Grammar
Non‑Restrictive Relative Clause
A non‑restrictive relative clause adds extra, non‑essential information about a noun and is always set off by commas. Removing the clause does not change the core meaning of the sentence. For example, “Students, who studied, passed” implies that all students studied. Non‑restrictive clauses comment on, rather than define, the noun.





549
Grammar
Comparative Clause
A comparative clause expresses comparison and often follows words like than or as. For example, “The fix was more complex than we expected.” These clauses frequently omit repeated material, relying on context for interpretation. Despite their reduced form, they are still full clauses structurally.





550
Grammar
Complement Clause
A complement clause is any clause required by a verb, adjective, or noun to complete its meaning. This includes many noun clauses and infinitival clauses. For example, “He insisted that the change be documented.” The clause is not optional and is selected by the governing word, making it part of the sentence’s core structure.





551
Grammar
Conditional Clause
A conditional clause expresses a condition under which the main clause holds. It is typically introduced by if, unless, or provided that. For example, “If the test fails, the deployment will be blocked.” Conditional clauses describe hypothetical or contingent situations and are a subtype of adverbial clauses.





552
Grammar
Copular (Linking) Verb
A copular verb connects the subject of a sentence to a subject complement, such as a predicate adjective or predicate noun, rather than expressing an action. The verb’s role is relational, not event‑based. While be is the most common copula, verbs like seem, appear, become, feel, taste, and prove can function as copular verbs depending on context. Compare “The soup tasted awful” (copular) with “She tasted the soup” (action).





553
Grammar
Free (Headless) Relative Clause
A free relative clause functions as a noun phrase even though it looks like a relative clause. It has no explicit antecedent noun. For example, “What she said surprised everyone” uses what she said to mean “the thing that she said.” These clauses commonly begin with what, whoever, whatever, or wherever and can serve as subjects or objects.





554
Grammar
Predicate Adjective (Predicative Adjective)
A predicate adjective is an adjective that follows a copular (linking) verb and describes the subject of the sentence rather than performing an action. It does not modify a noun directly; instead, it completes the meaning of the subject through the verb. Common copular verbs include be, seem, become, appear, remain, taste, and prove. For example, in “The plan was effective,” effective is a predicate adjective describing the plan.

The default join kind is innerunique, not inner. It silently deduplicates the left side's join keys before matching — you can lose rows you expected. Always specify kind= explicitly:

    kind=fullouter — keep unmatched rows from both sides
    kind=leftanti — left rows with no match on right (great for "not in")

Azure Compute Gallery and Disk Architecture
Default DiskControllerTypes when omitted 
If you don't specify DiskControllerTypes at definition creation, Azure assigns SCSI-only by default. This is why custom-captured images frequently can't deploy to v6/v7 SKUs even when nothing about the OS would prevent it.





762
Azure Compute Gallery and Disk Architecture
Sysprep /generalize — purpose 
Strips machine-specific identifiers (SID, computer name, drivers tied to specific hardware enumerations, activation tickets) so the image can deploy onto many new VMs without collisions. Does not compress, encrypt, or update.





763
Azure Compute Gallery and Disk Architecture
vTPM Virtual Trusted Platform Module. 
Provides measured boot — a tamper-evident log of what code ran during boot, signed by the TPM. Used for attestation: a remote service can verify a VM booted clean. Not a network or disk encryption feature, despite being lumped under "security."





764
Azure Compute Gallery and Disk Architecture
Managed image vs. Gallery image version 
Managed image (Microsoft.Compute/images) = older single-region, unversioned, no replication, no sharing. 
Gallery version (Microsoft.Compute/galleries/images/versions) = modern, versioned, replicated, shareable. 
Managed images still appear as intermediate capture targets (the --managed-image flag on az sig image-version create). Don't end-state on a managed image.





765
Azure Compute Gallery and Disk Architecture
Where can a new image version's source come from? 
A managed image, a snapshot, a VM, another existing image version, or a VHD. Cannot use a VM you don't have access to. Each source carries its own metadata that may or may not be compatible with your destination definition's feature flags.





766
Azure Compute Gallery and Disk Architecture
Why v6/v7 SKUs went NVMe-only 
The underlying AMD EPYC (and equivalent Intel) hosts wire physical NVMe SSDs directly to the CPU over PCIe. Virtualizing SCSI on top would mean NVMe→SCSI→NVMe translation, capping per-vCPU IOPS at a fraction of the silicon's potential. Microsoft cut SCSI on these SKUs to deliver the IOPS modern workloads expect.





767
Azure Compute Gallery and Disk Architecture
DiskControllerTypes = "SCSI, NVMe" 
A compatibility declaration — the image is fit to deploy on hosts of either controller type. The platform picks the right one at deploy time based on the destination SKU. Does not mean two disks are attached or that conversion happens.
