Endianness
Endianness describes the byte order used to store multi‑byte values in memory. Little‑endian systems store the least significant byte first (common on x86), while big‑endian systems store the most significant byte first. Misunderstanding endianness leads to reading correct bytes but interpreting the wrong number.

Byte Offset
A byte offset is the zero‑based position of a byte within a file or memory buffer. Offsets are how record‑oriented systems locate fields without delimiters, using fixed positions instead of searching.
........
Padding
Padding refers to unused bytes inserted between fields or records to satisfy alignment requirements or fixed‑width layouts. Padding bytes often contain 0x00, 0x20 (space), or 0xFF and should not be interpreted as data.
........
Alignment
Alignment is the requirement that certain values begin at specific byte boundaries (for example, 2‑byte or 4‑byte boundaries). Alignment improves performance but introduces padding that can confuse naive parsing.
........
Sentinel Value
A sentinel value is a special constant used to mark unused, deleted, or invalid data (such as 0xFFFF). Sentinel values repeat frequently and indicate structure, not content.
........
Fixed‑Length Record
A fixed‑length record is a data row that always occupies the same number of bytes. This allows record boundaries to be computed mathematically instead of discovered dynamically.
........
Record Stride
Record stride is the number of bytes you advance to move from one logical record to the next when scanning a file. Stride is often equal to record length but may differ due to page headers or interleaving.
........
Page
A page is a fixed‑size block of data (commonly 4 KB) used by the operating system and database engines for I/O and caching. Files are often processed page‑by‑page, not record‑by‑record.
........
Page Header
A page header is metadata stored at the beginning of a page that describes the page’s contents (type, record count, free space). Treating page headers as data produces misleading numeric patterns.
........
Memory‑Mapped File (mmap)
A memory‑mapped file maps file contents directly into a process’s address space so it can be accessed like an array. The operating system loads pages on demand instead of reading the entire file at once.
........
Demand Paging
Demand paging is the mechanism where file pages are loaded into memory only when accessed. This is why mmap can handle very large files efficiently without loading them fully into RAM.
........
Lazy Loading
Lazy loading means data is not read until it is actually needed. With mmap, touching a byte causes the OS to load the page containing it, hiding I/O behind memory access.
........
Binary Carving
Binary carving is the practice of scanning raw bytes for recognizable patterns (strings, integers, repetition) to infer structure when no schema is available.
........
Heuristic
A heuristic is an informed guess used when exact structure is unknown, such as scanning for repeated u16 values at consistent offsets to locate identifiers.
........
Structure vs Encoding
Structure defines where data lives (offsets, lengths, records). Encoding defines how values are represented (ASCII, UTF‑16, integers). Correct encoding is useless without correct structure.
........
ASCII‑Printable Range
The ASCII‑printable range (0x20–0x7E) represents visible characters. String carving relies on contiguous bytes in this range, which fails when padding or encoding breaks contiguity.
........
UTF‑16LE
UTF‑16LE encodes characters as two bytes (character byte + 0x00). This encoding breaks simple ASCII string carving and requires different detection logic.
........
False Positives (Binary Analysis)
A false positive occurs when repeated values look meaningful but are actually padding, flags, or headers. Repetition alone does not imply semantic meaning.
........
Invariant
An invariant is a value or pattern that remains consistent across pages or records. In binary analysis, invariants are used to identify structure boundaries.
........
Sampling
Sampling is the practice of examining small, repeated portions of a file (for example, the first few pages) to infer structure without parsing the entire file.
